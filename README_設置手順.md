# 写しこみ看板 v1.02 ― 設置手順

## 中身（8ファイル）

```
index.html              アプリ本体（名前を変えないでください）
sw.js                   圏外でも開くための仕組み
qrcode.min.js           QRを作る部品（同梱にしました）
manifest.json           ホーム画面用の設定
icon-192.png            アイコン
icon-512.png            アイコン
icon-512-maskable.png   Android用アイコン
apple-touch-icon.png    iPad・iPhone用アイコン
```

**この8つを、リポジトリ `utusikomi` の直下に置いてください。** フォルダは作りません。

URL は変わりません → `https://sirpromenadeendo-source.github.io/utusikomi/`

---

## v1.01 から変わったこと

### 1. 圏外でもアプリが開くようになりました

これまでは、一度開いてから**10分**すぎて圏外になると、**アプリ自体が開きませんでした**。
`sw.js` を入れたことで、圏外でも開けます。

### 2. QRを作る部品を同梱しました

これまでインターネット（cdnjs）から読んでいたものを、アプリに含めました。
cdnjsが落ちても、社内ネットワークが塞いでいても、影響を受けません。

### 3. 設定タブに「アプリの状態」を追加しました

いまの版数、圏外対応が効いているか、オンラインかどうかが分かります。
その下の「**🔄 最新に更新**」は、動きがおかしいときの脱出口です。
手元の控えを捨ててサーバから読み直します。**記録は消えません。**

---

## 更新は今までより速くなります

`sw.js` は「**通信を優先し、圏外のときだけ手元の控えを使う**」作りです。

| | v1.01まで | v1.02 |
|---|---|---|
| 更新の反映 | 最大10分待つ | すぐ |
| 圏外で開く | ✕ | ○ |

以前この仕組みを試してやめた原因は「控えを先に見ていたため、更新が反映されない」ことでした。
今回は順番が逆です。**`sw.js` の中の順番（通信 → だめなら控え）を入れ替えないでください。**

---

## やめたくなったときは（重要）

**`sw.js` をサーバから消すだけでは、端末に残り続けます。**（実測で確認済み）

正しい手順は「**消す**」ではなく「**差し替える**」です。

1. 別途お渡ししている `sw_kill.js` の中身を、`sw.js` に上書きする
2. そのままGitHubに置く
3. 各端末がアプリを**2回**開けば、登録も控えも消え、v1.01相当の状態に戻る

全端末に行き渡ったら、`sw.js` と、`index.html` 内の `navigator.serviceWorker.register` の行を消してかまいません。

**注意**: ずっと圏外に置きっぱなしの端末には届きません。一度オンラインで開いてもらう必要があります。

## 動作の条件

- **Android**: 7 以上 ／ Chrome 108 以上
- **iPad・iPhone**: Safari でホーム画面に追加
- `file://`（ファイルを直接開く）場合、圏外対応は働きませんが、アプリは通常どおり動きます

## 変わっていないこと

QRの中身、看板の見た目、記録、CSV／JSONの形式 ― すべて v1.01 と同じです。
アプリ②（写真台帳照合アプリ）との連携も、これまでどおりです。
