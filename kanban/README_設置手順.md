# 写しこみ看板 v1.01 ― ホーム画面に追加して、ブラウザのバーを出さずに開く

## このフォルダの中身

```
kanban/                     ← このフォルダごとGitHubに置きます
├ index.html                ← アプリ本体（名前を変えないでください）
├ manifest.json             ← ホーム画面用の設定
├ icon-192.png              ← アイコン
├ icon-512.png              ← アイコン
├ icon-512-maskable.png     ← Android用（丸や角丸に切り抜かれても欠けない版）
└ apple-touch-icon.png      ← iPad・iPhone用アイコン
```

**Service Worker は使っていません。** つまりアプリを端末にキャッシュしません。
以前この方式を試してやめた「更新が反映されない」問題は起きません。
更新は今までどおり、GitHub Pages の通常のキャッシュ（10分ほど）で反映されます。

## 設置のしかた

### 1. フォルダごと置く

リポジトリの中に `kanban` などのフォルダを作り、**この6ファイルをまとめて**入れてください。

```
（リポジトリ直下）
├ kanban/
│  ├ index.html
│  ├ manifest.json
│  └ icon-*.png
└ （ほかのアプリ…）
```

URLは `https://＜ユーザー名＞.github.io/＜リポジトリ名＞/kanban/` になります。

### 2. なぜフォルダに分けるのか

`manifest.json` の `scope`（アプリの適用範囲）を、このフォルダの中だけに限るためです。
リポジトリ直下に置くと、同じ場所にある他のアプリまで「写しこみ看板の一部」と
みなされてしまいます。

### 3. ファイル名を変えたい場合

`index.html` のままにしてください。`manifest.json` の `start_url` が `"./"`（＝フォルダ）
になっているため、`index.html` 以外の名前だと起動できません。

どうしても別名にする場合は、`manifest.json` の2か所を直してください。

```json
"start_url": "./看板.html",
"scope": "./",
```

## 使い方（Android）

1. Chrome で `.../kanban/` を開く
2. 右上の「⋮」メニューを開く
3. **「アプリをインストール」** を選ぶ

これでホーム画面にアイコンができ、**ブラウザのバーが出ない状態**で開きます。

### 「アプリをインストール」が出ないとき

代わりに「ホーム画面に追加」しか出ない場合、それは**ただのショートカット**なので、
ブラウザのバーが出たままになります。次を確認してください。

- **httpsで開いているか**（GitHub Pagesならhttps）
- **`.../kanban/` で開いているか**（`index.html` まで打っても可）
- **manifest.json とアイコンが同じフォルダにあるか**
- 一度ページを操作してから（数秒たってから）メニューを開く

うまくいかないときは、Chromeのアドレス欄に `chrome://inspect` … ではなく、
パソコンのChromeで同じURLを開き、**F12 →「アプリケーション」→「マニフェスト」**を
見るとエラーが表示されます。

## 使い方（iPad・iPhone）

1. Safari で `.../kanban/` を開く
2. 共有ボタン（□に↑）→ **「ホーム画面に追加」**

iOSもバーが出ない状態で開きます。

**注意**: これまでiPadのホーム画面に追加していた場合、古いアイコンは
そのままでは切り替わりません。一度削除して、追加し直してください。

## 元に戻したいとき

`index.html` の `<head>` にある次の行を消せば、元の（バーが出る）動きに戻ります。

```html
<link rel="manifest" href="manifest.json">
<meta name="apple-mobile-web-app-capable" content="yes">
```
